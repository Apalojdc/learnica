Analyse et amélioration de la complexité

L'optimisation d'un algorithme vise à améliorer ses performances en termes de temps d'exécution et/ou d'utilisation mémoire. Voici les techniques courantes :

Taille des données (n)
Temps d'exécution
O(1)
O(log n)
O(n)
O(n²)
O(2^n)

Stratégies d'optimisation

  • Éviter les calculs redondants : mémorisation, précalculs, etc.
  • Choisir les structures de données adaptées : tableaux, listes chaînées, arbres, tables de hachage, etc.
  • Simplifier les expressions mathématiques : factorisation, réduction, etc.
  • Optimiser les boucles : déroulement, fusion, etc.
  • Éliminer les calculs inutiles : arrêt anticipé, filtrage préalable, etc.

Compromis temps-espace

L'optimisation d'un algorithme implique souvent un compromis entre le temps d'exécution et l'espace mémoire utilisé.

Principe de base du compromis temps-espace :

  • Gagner du temps en utilisant plus de mémoire (ex : mémorisation)
  • Économiser de la mémoire au prix d'un temps d'exécution plus long
Exemple : Calcul du n-ième nombre de Fibonacci

// Approche récursive (économe en espace mais lente)
fonction fib_recursif(n)
    si n ≤ 1 alors retourner n
    sinon retourner fib_recursif(n-1) + fib_recursif(n-2)
fin fonction

// Approche dynamique (plus rapide mais utilise plus de mémoire)
fonction fib_dynamique(n)
    tableau memo[0..n]
    memo[0] ← 0
    memo[1] ← 1
    pour i de 2 à n faire
        memo[i] ← memo[i-1] + memo[i-2]
    fin pour
    retourner memo[n]
fin fonction

Profiling et benchmarking

Pour optimiser efficacement un algorithme, il est essentiel de mesurer ses performances et d'identifier les goulots d'étranglement.

Méthodes d'évaluation

  • Profiling : analyse détaillée de l'exécution d'un programme pour identifier les parties les plus coûteuses
  • Benchmarking : comparaison des performances de différentes implémentations
  • Analyse asymptotique : étude théorique du comportement de l'algorithme pour de grandes entrées

Règle d'optimisation pragmatique :

  1. Faites fonctionner l'algorithme correctement
  2. Mesurez ses performances
  3. Optimisez uniquement les parties critiques

Ne tombez pas dans le piège de l'optimisation prématurée !

Paradigmes de conception algorithmique

Les paradigmes de conception algorithmique sont des approches générales pour résoudre des problèmes. Ils fournissent un cadre méthodologique pour élaborer des solutions efficaces.

Diviser pour régner

Ce paradigme consiste à décomposer un problème en sous-problèmes plus petits, à résoudre ces sous-problèmes, puis à combiner leurs solutions pour résoudre le problème initial.

Exemple : Tri fusion (Merge Sort)

fonction tri_fusion(tableau)
    si longueur(tableau) ≤ 1 alors
        retourner tableau
    fin si

    // Diviser
    milieu ← longueur(tableau) / 2
    gauche ← tableau[0..milieu-1]
    droite ← tableau[milieu..fin]

    // Régner (appels récursifs)
    gauche_triée ← tri_fusion(gauche)
    droite_triée ← tri_fusion(droite)

    // Combiner
    retourner fusion(gauche_triée, droite_triée)
fin fonction

Complexité du tri fusion :

  • Temps : O(n log n)
  • Espace : O(n)

Programmation dynamique

Cette approche consiste à résoudre des problèmes complexes en les décomposant en sous-problèmes plus simples et en stockant les résultats intermédiaires pour éviter les calculs redondants.

Étapes de la programmation dynamique :

  • Identifier si le problème peut être décomposé en sous-problèmes qui se chevauchent
  • Définir une relation de récurrence entre les sous-problèmes
  • Stocker les résultats des sous-problèmes (mémorisation)
  • Construire la solution finale à partir des résultats stockés
    • Exemple : Problème du sac à dos (Knapsack Problem)

    fonction knapsack(poids[], valeurs[], n, capacité)
        dp[0..n][0..capacité] ← tableau de zéros

        pour i de 1 à n faire
            pour w de 1 à capacité faire
                si poids[i-1] ≤ w alors
                    dp[i][w] ← max(valeurs[i-1] + dp[i-1][w-poids[i-1]], dp[i-1][w])
                sinon
                    dp[i][w] ← dp[i-1][w]
                fin si
            fin pour
        fin pour

        retourner dp[n][capacité]
    fin fonction

    Algorithmes gloutons

    Les algorithmes gloutons font, à chaque étape, le choix localement optimal dans l'espoir d'aboutir à une solution globalement optimale.

    Attention : Les algorithmes gloutons ne garantissent pas toujours la solution optimale globale. Ils sont efficaces pour certains problèmes spécifiques où le choix local optimal mène effectivement à l'optimum
